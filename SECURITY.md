# Security Policy

## Supported Versions

Only the latest version of the Forex Trading Bot is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

- Do not create a public GitHub issue for security vulnerabilities
- Contact us directly at [your-email@example.com] (replace with actual contact)
- Provide detailed information about the vulnerability
- Include steps to reproduce the issue
- Allow reasonable time for a fix to be implemented before disclosure

## Security Best Practices

When using this trading bot, please follow these security best practices:

- Never hardcode API keys or credentials in the code
- Use a separate trading account with limited funds for testing
- Monitor your trading activity regularly
- Keep your MT5 terminal and operating system updated
- Use secure networks when connecting to trading platforms
- Regularly review and audit the code before deployment

## Known Security Considerations

- This bot executes real trades on financial markets - use with caution
- The bot stores trade history locally - protect these files appropriately
- Network connections to MT5 may be intercepted - use secure connections
- Always verify the code before running it with real money

## Dependencies

We strive to keep dependencies minimal and up-to-date. If you find security issues in our dependencies:

1. Check if there's an updated version available
2. Report the issue to the respective dependency maintainers
3. Notify us so we can update our requirements
